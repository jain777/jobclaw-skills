# Per-question quality bar — prep-interview

A good prep question:

1. **Is plausible.** A real interviewer at that role / level would actually ask this.
2. **Earns its slot.** Either:
   - tests a competency the role actively needs (per `jobSummary.dayToDayResponsibilities`), OR
   - probes a claim the user has made (a top experience bullet, a flagship project), OR
   - opens a strategic conversation tied to recent company news.
3. **Has a useful STAR.** The talking-point bullet cites a real Situation / Task / Action / Result the user actually lived. If you can't find that in the profile, drop the question and pick another.
4. **Is one question, not three.** "Tell me about a time you [X] and how [Y] and what [Z]" is three questions — split or trim.
5. **Has a round tag.** One of the canonical rounds from `mock-interview/reference/report-schema.md`. No orphan questions.

## Anti-patterns at the question level
- **Brain-teasers** ("how many golf balls fit in a 747") — almost no serious loop uses these in 2026.
- **Stack-rank trivia** ("difference between TCP and UDP") for senior roles.
- **"Tell me about yourself"** stacked at the top of multiple rounds — one is fine in a recruiter screen.
- **Questions that require data the user doesn't have** ("our Q3 LTV moved 12% — why?").
- **Hostile or trick-style framings** — interviewers asking those are red-flag interviewers, not a pattern to prep for.

## Mix targets per round bucket

| Round bucket                       | Generic | Company-specific | Profile-targeted |
|------------------------------------|---|---|---|
| recruiter-screen / behavioral      | 2–3 | 1   | 1–2 |
| technical / case / portfolio       | 1   | 1–2 | 1–2 |
| strategy / hiring-manager / final  | 1   | 2–3 | 1–2 |

Adjust ±1 per round; total stays 8–12.

## STAR bullet shape

Each STAR bullet is **one fragment** — the user fills the rest in voice:

- *S* — *"Stripe developer onboarding, time-to-first-API-call sat at 12 min in 2023"*
- *T* — *"reduce TTFAC to compete with serverless onboarding"*
- *A* — *"reshaped the getting-started path; cut six steps; partnered with API team on a programmatic auth helper"*
- *R* — *"40% reduction, 12→7 min; documented in the quarterly review"*

Don't write the full STAR essay — the user will narrate. The bullet pre-loads the *content*.
